# OpenTeams 5 篇文章 ASCII 配图方案

说明：以下方案统一采用横向流程图 + 卡片框体风格，建议在 Markdown 中用 ```text 代码块直接嵌入。每篇至少 1 张主图，均围绕多智能体协作、会话流、执行记录、技能生态等主题设计。

## 1. 产品价值主题：把单兵 AI 变成协作型 AI 团队

### 图 1-1：多智能体协作总览

用途：适合放在文章开头，快速建立 OpenTeams 的产品心智。

```text
+------------------+      +------------------+      +------------------+
|     Product      | ---> |    Researcher    | ---> |   Tech Writer    |
|      Agent       |      |      Agent       |      |      Agent       |
+------------------+      +------------------+      +------------------+
          \                         |                         /
           \                        v                        /
            \            +--------------------+             /
             ----------> |    Shared Session   | <----------
                         |  messages + context |
                         +--------------------+
                                   |
                                   v
                         +--------------------+
                         |   Final Deliverable |
                         |   docs/ + records   |
                         +--------------------+
```

建议图注：OpenTeams 让多个角色型 Agent 在同一会话中协作推进同一项交付，而不是各自孤立输出。

## 2. 协作场景主题：一次会话如何驱动多人机共同完成任务

```text
+--------+      +----------+      +------------+      +---------+
|  User  | ---> |  Editor  | ---> |  @mention  | ---> | Agents  |
+--------+      +----------+      +------------+      +---------+
                                                        |  |  |
                                                        v  v  v
                                                   +--------------+
                                                   | Shared Thread |
                                                   | facts + drafts|
                                                   +--------------+
                                                          |
                                                          v
                                                   +--------------+
                                                   |Merged Output  |
                                                   |article / plan |
                                                   +--------------+
```

建议图注：用户只需要发起一次任务，会话中的分工、提及、汇总和回写都在统一上下文里完成。

## 3. 技术亮点主题：上下文同步、执行轨迹与可追溯记录

### 图 3-1：执行记录与上下文快照

用途：适合放在“架构能力”或“为什么可追踪”部分。

```text
+----------------+      +----------------+      +----------------+
| Chat Message   | ---> | Agent Run      | ---> | Run Directory  |
| mentions/meta  |      | index + state  |      | logs / diff    |
+----------------+      +----------------+      +----------------+
         |                       |                        |
         v                       v                        v
+----------------+      +----------------+      +----------------+
| messages.jsonl |      | context.jsonl  |      | untracked view |
| session memory |      | run snapshot   |      | output records |
+----------------+      +----------------+      +----------------+
```

建议图注：从消息、运行到日志与差异文件，OpenTeams 将协作过程拆成可检查、可回放、可归档的执行轨迹。

## 4. 团队效率主题：减少切换成本，让任务并行推进

### 图 4-1：串行工作 vs 并行协作

用途：适合放在效率对比或价值说明部分。

```text
Traditional:
[Plan] ---> [Research] ---> [Write] ---> [Review] ---> [Publish]

OpenTeams:
            +------------ Researcher ------------+
[Goal] ---> +------------ Tech Writer -----------+ ---> [Editor Merge] ---> [Publish]
            +------------ Marketing -------------+
            +------------ Reviewer --------------+
```

建议图注：传统流程依赖单线程切换，OpenTeams 更适合把可拆分任务同时交给多个 Agent 并最终集中收口。

## 统一使用建议

- 全部图片建议使用 `text` 代码块，避免不同 Markdown 渲染器挤压对齐。
- 正文首次出现时可配一行图注；如果文章较长，可在段中重复使用同一风格的小图。
- 若需要增强统一性，可统一在图前使用格式：`图 X：xxxx`。
- 如版心较窄，可优先保留框体与箭头，不必强行保留所有说明文字。
